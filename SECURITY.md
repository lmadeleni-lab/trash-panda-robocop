# Security Policy

## Supported Scope

This repository is an early-stage open-source starter kit. Security and safety reports are both welcome, especially for:

- API exposure or auth mistakes
- trusted-network or local-LAN boundary bypasses
- unauthorized access to future camera-control or sensor-control surfaces
- unsafe actuation paths
- bypasses around human/pet exclusion
- unsafe hardware assumptions
- privacy issues around image capture or stored event data

## Reporting

Please do not open a public issue for vulnerabilities or safety bypasses that could be abused.

Instead, contact the maintainer privately and include:

- a short summary
- affected file or module
- reproduction steps
- risk level and likely impact
- any suggested fix or mitigation

If private reporting channels are not yet set up, create a minimal GitHub issue without exploit details and ask for a secure contact path.

## Response Goals

- acknowledge reports quickly
- confirm severity and scope
- ship a fix or mitigation when practical
- document public remediation after sensitive details are no longer risky to disclose
